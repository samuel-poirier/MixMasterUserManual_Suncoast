### Information visible on this page:

#### General

- **SKU**: Product SKU number
- **UPC**: Universal Product Code for the case
- **UPC Ind. Unit**: UPC for the individual sellable unit within the case (if different from the case UPC)
- **Reference Name**: Reference name for the product
- **Name**: Product display name
- **Category**: Product category (e.g., Promotional item)
- **Un./Case**: Number of individual units per case
- **Un./Layer**: Number of units per pallet layer
- **Un./Pallet**: Total number of units per pallet
- **Case grouping unit (Default: 1)**: Some products are sold as grouped units. This field is for defining how many physical cases make up one sellable unit. MixMaster then calculates distribution and palletization accordingly.
- **Product Family**: This field, which is used only when the associated checkbox is enabled in the [Extra Details](#extra-details) section, is used by the Palletizer for better handling of the cases in the Row-Builder.
- **Retired**: Whether the product has been retired from production
- **Do not sub.**: Whether the product should not be substituted with another product that shares its UPC
- **Need superior wrap program**: Whether the product requires a special wrap program (double wrapping). If any cases with this option enabled are found on a pallet, the entire pallet will be wrapped with the special wrap program.

#### InnoPick Configurations

- **InnoPick Compatible**: Whether the product can be assigned to InnoPick
- **Allow replenishments**: Whether the system can call depalletizing replenishments for this product
- **Use Manual Induction**: Whether the product must be assigned to Manual Induction instead of Depalletizers

#### Dimensions

- **Length / Width / Height**: Case dimensions in mm
- **Layer length / Layer width**: Layer dimensions in mm
- **Gross Weight**: Case weight in grams
- **Volume**: Case volume in ml
- **Case Dimension / Weight and Layer Dimension Tolerance**: Allowable variance in dimensions (+/- mm or g). If set to 0, default tolerances will be used.
- **Box Top Width / Box Top Length**: Top face dimensions in mm
- **Height For Robots**: Height value used by robotic systems in mm, if a different height than the product height is needed. If set to 0, the product height is used.

### Extra Details

<img src="../images/truckloads/product_extradetails.png" width="500" alt="Product Details Pop-up" />

**Explanations:**

- **Force 4 layers per replenishment (replenishment quantity may use a second lane)**: Used for high-movers to force larger replenishments, with the understanding that this can cause a replenishment to fill up more than one InnoPick lane.
- **Keep close to descrambler (used for high-volume or difficult-to-clamp products)**: When enabled, MixMaster will request this product to be kept close to the descrambler deposit station for better efficiency when depalletizing.
- **Use product family field for palletizer instructions instead of dimension-based algorithm**: When enabled, MixMaster will use the Product Family designation from the [General](#general) Section. This is used by the Palletizer for better handling of the cases in the Row-Builder.

### Recovery Actions

<img src="../images/configuration/recovery-actions.png" width="600" alt="Recovery Actions Options" />

#### Send product details to InnoPick


- This recovery action makes MixMaster send product info for this SKU to InnoPick.
    - MixMaster normally sends all product info to InnoPick at the start of every truckload launch, so this recovery action is only needed if there is a new product that the operators want to configure in a specific way inside InnoPick Manager.
    - With this feature, the product could be sent to InnoPick Manager and then the user could configure the product to be forced to a certain level or have a certain minimum replenishment size, or some other specific configuration.
