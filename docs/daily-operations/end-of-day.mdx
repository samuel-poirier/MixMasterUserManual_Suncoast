# End of Day

**[Home](../index.md) > [Daily Operations](index.md) > End of Day**

## Overview

Proper end-of-day procedures ensure that production is accounted for, the system is left in a clean state, and the next shift has the information they need.

## Verify Production Status

Before ending the shift, check the following:

1. **Truckload Status**
    - Navigate to the [Truckloads](../truckloads/index.md) page
    - Verify that all launched truckloads have reached **Completed** status
    - If any truckloads are still **In Progress**, coordinate with supervision about whether to wait for completion or hand off to the next shift

2. **Replenishment Status**
    - Check the [Replenishments](../replenishments.md) page
    - Verify that all replenishments have been completed
    - Note any pending or active replenishments and look into why they are not completed

3. **Production Summary**
    - Review the [Production Summary](../home-page/production-summary.md) for final counts
    - Confirm completed numbers match expectations

---

## Review Reports

At the end of production, review the following reports for any issues that should be documented or escalated:

-   **[Faults History](../reports/faults-history.md)**: Note any recurring faults during the shift
-   **[Out of Stock](../reports/out-of-stock.md)**: Review products marked OOS — these may need replenishment before the next production run
-   **[Automation Performance](../reports/automation-performance.md)**: Review production performance and note any items or events that need to be followed up on.
